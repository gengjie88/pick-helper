# PickHelper - 英雄联盟客户端辅助工具

一个基于 Electron 开发的英雄联盟客户端辅助工具，支持自动接受对局和大乱斗自动选英雄功能。

## 功能特性

### 核心功能
- ✅ **自动接受对局** - 匹配成功后自动点击接受
- ✅ **大乱斗自动选英雄** - 根据优先级列表自动交换备选池最高优先级英雄

### 其他特性
- 🎨 简洁现代的浅色 UI 界面
- 📱 双页面设计（主页 + 设置页）
- 🔍 英雄搜索添加，支持中文名搜索
- 📊 实时显示客户端状态和游戏阶段
- 📝 运行日志实时查看
- 💾 设置自动保存
- 🔄 每日自动从 Riot Data Dragon 更新英雄数据
- 🖱️ 最小化到系统托盘后台运行

## 安装与运行

### 环境要求
- Node.js >= 16.0.0
- Windows 系统（LCU API 仅支持 Windows 英雄联盟客户端）

### 安装依赖
```bash
cd pick-helper
npm install
```

### 运行
```bash
# 普通运行
npm start

# 开发模式（打开开发者工具）
npm run dev
```

## 使用说明

### 主页面
- **客户端状态**：显示是否已连接到英雄联盟客户端
- **游戏阶段**：显示当前游戏阶段（大厅/匹配中/选角中等）
- **预选英雄优先级**：查看已配置的英雄优先级列表
- **数据更新时间**：显示英雄数据版本和更新日期

### 设置页面
- **功能设置**：
  - 自动接受对局（默认开启）
  - 大乱斗自动选英雄（默认开启）
- **英雄优先级配置**：
  - 搜索框输入英雄名称搜索
  - 点击搜索结果添加到优先级列表
  - 点击删除按钮移除英雄
- **英雄数据**：查看数据版本，手动刷新
- **运行日志**：实时查看工具运行日志

## 工作原理

### LCU API 通信
- 通过检测 `LeagueClientUx.exe` 进程获取 LCU API 的端口和认证令牌
- 使用 HTTPS 基本认证方式调用 LCU API
- 忽略自签名证书验证（LCU 使用自签名证书）

### 自动接受对局
- 轮询 `/lol-matchmaking/v1/ready-check` 接口
- 检测到 `state: "InProgress"` 时自动调用接受接口

### 大乱斗自动选英雄
- 检测游戏阶段为 `ChampSelect` 且模式为 `ARAM`
- 获取选角会话数据，提取备选池英雄列表
- 比较当前手持英雄和备选池英雄的优先级
- 当备选池有更高优先级英雄时，自动执行交换操作
- 包含去重机制，避免交换冷却期重复请求

### 英雄数据更新
- 启动时检查并更新英雄数据
- 每日自动检查更新（每小时检测跨天）
- 从 Riot Data Dragon API 获取最新英雄信息
- 数据本地持久化存储

## 注意事项

1. **仅支持 Windows**：LCU API 仅在 Windows 平台的英雄联盟客户端上可用
2. **需要管理员权限**：检测进程可能需要管理员权限
3. **网络要求**：首次运行需要联网下载英雄数据
4. **封号风险**：使用第三方工具存在一定风险，请自行评估使用

- **electron-store** - 数据持久化存储
- **原生 HTTPS** - LCU API 通信
- **原生 HTML/CSS/JS** - 前端界面

## 预设（Presets）说明

项目支持在设置页面中管理多个英雄优先级预设（分组）。预设会保存在本地配置文件中，默认路径（Windows）为：

- C:\\Users\\<你的用户名>\\AppData\\Roaming\\PickHelper\\config.json

在该 JSON 文件中，与预设相关的字段位于 `settings` 下：

- `settings.presets`：预设数组，每项示例 `{ id: "preset_...", name: "名称", heroPriority: [22,48,30] }`
- `settings.selectedPreset`：当前选中的预设 id（字符串）
- `settings.heroPriority`：当前正在使用的英雄优先级数组（会随选中预设同步）

通过应用 UI 管理预设：
- 在 **设置** 页面的预设下拉选择当前分组，切换后会立即同步到主页面并应用对应的优先级列表。
- 点击 **新建** 会清空编辑区（进入编辑新预设），编辑完成后填写名称并点击 **保存** 将生成一个新的预设并自动选中。
- 若在编辑时未改变名称则点击 **保存** 会覆盖当前选中预设。
- 点击 **删除** 会弹出确认，删除后会自动切换到第一个预设；当仅剩一个预设时禁止删除。

查看配置文件（PowerShell 示例）：
```powershell
Get-Content "$env:APPDATA\\PickHelper\\config.json" | Out-String
```

注意：直接编辑 `config.json` 前请关闭应用以避免写入冲突；建议通过应用内的“导入/导出”功能（若需要）来备份与恢复预设。
